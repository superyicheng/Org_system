# 3-minute demo script

## 0:00–0:20 — Hook

“Teams document successful systems. They rarely preserve failed experiments in a form that can stop the next engineer from repeating them. Hive.skill turns both fixes and failures into executable team memory.”

## 0:20–0:55 — Veteran distills a fix

1. Select **Veteran**.
2. Type the veteran prompt from `demo-prompts.txt`.
3. Say: “Nothing is prefilled. The veteran describes what actually worked.”
4. Click **Save to Team Hive**.
5. Say: “Hive extracts the fingerprint, assumptions, and executable fix, then persists it to ChromaDB.”

## 0:55–1:50 — New hire proposes an expensive idea

1. Select **New Hire**.
2. Type the 8-TB log-vectorization prompt.
3. Press Enter.
4. Say: “Before any GPU or Kubernetes Job is created, Hive searches both successful and failed team knowledge.”
5. Point to the real vector similarity, prior cost, and evidence comparison.
6. Say: “The prior run consumed 148 GPU hours for only a 3% accuracy gain.”

## 1:50–2:30 — Execute the safer plan

1. Switch between **AI Evidence** and **Safe Script**.
2. Click **Block and adopt safe plan**.
3. Let the terminal animation finish.
4. Pause on the `148 GPUh → 6 GPUh` impact screen.

## 2:30–3:00 — Close

“Retrieval and AI reasoning are real. Execution is simulated for the hackathon, but the generated scripts are usable. Hive.skill does not only help engineers fix known errors—it prevents teams from paying for the same failed innovation twice.”

Final line:

> Hive.skill compiles team experience into an AI execution guard.
